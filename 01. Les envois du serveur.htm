1. Les envois du serveur

Comme l’a indiqué le cours, un serveur HTTP peut transmettre soit le contenu d’un fichier, soit le résultat de son exécution. Le premier exercice de cette séance examine d’abord le premier cas, avec de simples fichiers HTML déjà existants. Les exercices suivants s’occupent de pages HTML créées dynamiquement, en particulier en mettant en page les variables d’environnement communiquées par le serveur. Ainsi l’exercice 2 consiste à créer dynamiquement une page HTML minimale. L’exercice 3 crée une page en donnant le nom du script exécuté par le serveur, le nom du serveur et celui du client. Les exercices 4, 5, 6 et 7 s’occupent de créer des tables selon différentes présentations, en respectant néanmoins les règles d’accessibilité des sites Web afin que les mal-voyants puissent tout de même profiter de leurs informations.


1 Lecture d’un fichier HTML

Vous avez à votre disposition une version papier d’un extrait du fichier HTML correspondant au cours numéro 1 (voir l’annexe qui vous a été fournie). Ce fichier HTML peut être lu comme un simple fichier par votre navigateur, sans passer par un serveur HTTP.

  1. A quoi sert la première ligne ?
  2. Donner la structuration de ce texte et, d’une façon plus générale, lister les balises permettant de structurer un texte et celles permettant de mettre en forme ce texte. Comment, en HTML, écrit-on une liste ?
  3. Qu’est ce qui permet l’affichage de ce texte à l’écran et à l’impression.
  4. Où se situent ces informations sur l’affichage ?
  5. A quoi sert la première balise meta ?
  6. Relever les occurrences du caractère esperluète (&) ; à quoi servent-elles ?
Solution Télécharger lecture_html.txt 
1. Doctype indique la version de la grammaire HTML à utiliser

2. Il y a un en-tête (balise Head) et un corps (balise Body); 
le corps a deux balises Div, la première pour le titre de l'enseignement,
la deuxième pour le contenu proprement dit,
qui comporte une liste (balise Ul) de deux éléments (balise Li).

3. Les deux balises Link dont l'attribut Rel vaut "stylesheet".

4. Les attributs Href de ces deux balises indiquent où sont ces feuilles de styles,
elles sont fournies par le serveur Http de la licence.

5. A préciser l'encodage du texte, notamment les lettres accentuées.

6. Les occurrences sont &amp; &gt et &nbsp;
qui servent à afficher respectivement "&" ">" et " ". 
Les deux premiers sont des caractères réservés de XHTML
et doivent donc être transcodés quand on veut les afficher.
Le troisième est un espace dont on précise qu'il est insécable
(pas remplaçable par un saut de ligne).

2 Première page HTML créée dynamiquement

Tout envoi à un client HTTP commence par au moins un en-tête, suivi d’une ligne blanche, autrement dit suivi de deux sauts de lignes. L’en-tête obligatoire est Content-Type: (le deux-points doit être accolé) suivi d’un type indiquant le contenu, le plus fréquent étant text/html. Pour les types commençant par text, il est recommandé de préciser l’encodage de l’alphabet, le plus général étant l’Unicode sur 8 bits, ce qui donne : text/html; charset=utf-8.

Écrire un script Shell, qui produit une page HTML donnant la date à laquelle le script est exécuté par le serveur HTTP. On pourra utiliser la commande Shell echo et la commande Unix cat, suivi d’un double chevron ouvrant (<<) et d’une chaîne de caractère arbitraire pour envoyer du texte dans le flux de sortie.

Solution Télécharger html_date.txt 
#!/bin/sh
cat <<EOF
Content-Type: text/html; charset=utf-8

<html><head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title> Page HTML dynamique indiquant la date</title>
</head>
<body>
Voici la date d'aujourd'hui :
<br />
EOF
date
echo '</body></html>'

# Envoyer "<br />" permet d'afficher la date en dessous de la ligne
# "Voici la date d'aujourd'hui".
# Mettre une ligne vide dans le texte HTML 
# ne provoquerait pas l'affichage en deux lignes par le navigateur,
# car il identifie espace et saut de ligne.

3 Afficher les informations du serveur

Lorsqu’un serveur HTTP exécute des programmes calculant des pages HTML, il place dans l’environnement certaines variables, les plus importantes étant indiquées dans la tableau ci-dessous.

Variables d’environnement fournies par le serveur HTTP
NomSignification
SCRIPT_NAME nom du script
SERVER_ADMIN nom de l’administrateur du serveur
SERVER_NAME nom de l’hôte du serveur
SERVER_SOFTWARE nom du serveur
SERVER_ADDR adresse Internet du serveur
REMOTE_PORT port où se déroule la connexion
REMOTE_ADDR adresse Internet du client
HTTP_USER_AGENT le nom du client
HTTP_ACCEPT_LANGUAGE langage préféré du client (fr-fr pour le français académique)
REQUEST_URI requête HTTP envoyée par le client
QUERY_STRING suite des couples variable / valeur dans la requête

Écrire un script Shell renvoyant le nom par lequel le serveur HTTP l’exécute, et le nom du serveur et du client. Les trois noms seront écrits en gras, grâce à la balise strong et sa fermante. Par exemple :

je suis Custer servi par Apache pour Firefox.

Solution Télécharger custer.txt 
#!/bin/sh 
echo 'Content-Type: text/html; charset=utf-8'
echo
echo '<html><head>'
echo '<title> Afficher les informations connues du serveur</title>'
echo '<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />'
echo '</head>'
echo '<body>'
echo 'Je suis le script <strong>' 
echo $SCRIPT_NAME 
echo '</strong>'
echo ' servi par <strong>' $SERVER_SOFTWARE '</strong>'
echo ' pour <strong>' $HTTP_USER_AGENT '</strong>'
echo '</body></html>'

4 Table de quelques variables du serveur

On se propose ici de reproduire par un script Shell une variante de la table HTML figurant dans l’énoncé de l’exercice précédent. On se limitera à une table composée de cinq lignes et deux colonnes, où figurent les variables suivantes initialisées par le serveur (nom en première colonne, valeur en deuxième colonne) :

  • SERVER_NAME nom de l’hôte du serveur
  • SERVER_SOFTWARE nom du serveur
  • SERVER_ADDR adresse Internet du serveur
  • REMOTE_PORT port où se déroule la connexion
  • REMOTE_ADDR adresse Internet du client

On rappelle qu’en HTML, une table se déclare par la balise table et sa balise fermante, que chaque ligne se commence par la balise tr (tr = table row = rangée de tableau) et chaque cellule par la balise td (td = table data = données de tableau) ; th définit une cellule d’entête (th = table header = entête de tableau).

De plus, le W3C a publié de nombreux travaux concernant l’accessibilité des documents sur Internet pour permettre au plus grand nombre d’y accéder :

  • Web Accessibility Initiative (WAI)
  • Web Content Accessibility Guidelines 1.0
  • Introduction à l’accessibilité du Web

En ce qui concerne les tables HTML, il est exigé :

  • que les colonnes soient nommées au moins une fois, c’est-à-dire qu’il y ait au moins une ligne tr composée exclusivement de balises th , avec comme contenu ici "Noms des variables" et "Valeurs des variables" ;
  • qu’elles comportent une légende, grâce à la balise caption ;
  • qu’elles comportentr également un résumé grâce à l’attribut summary de la balise table.

Ecrire le script Shell demandé réalisant ces spécifications.

Solution Télécharger table_serveur.txt 
#!/bin/sh 
echo 'Content-Type: text/html; charset=utf-8'
echo
echo '<html><head>'
echo '<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />'
echo '<title> Table des variables du serveur</title>'
echo '</head>'
echo '<body>'
echo '<table summary="Table des principales variables du serveur">’ 
echo '<caption> Table des variables du serveur</caption>'
echo '<tr><th>Nom</th><th>Valeur</th></tr>'
echo '<tr><td>SERVER_NAME</td><td>'$SERVER_NAME'</td></tr>'
echo '<tr><td>SERVER_SOFTWARE</td><td>'$SERVER_SOFTWARE'</td></tr>'
echo '<tr><td>SERVER_ADDR</td><td>'$SERVER_ADDR'</td></tr>'
echo '<tr><td>REMOTE_PORT</td><td>'$REMOTE_PORT'</td></tr>'
echo '<tr><td>REMOTE_ADDR</td><td>'$REMOTE_ADDR'</td></tr>'
echo '</table>' 
echo '</body></html>'

5 La multiplication des TD

On souhaite écrire une page HTML présentant la table de multiplication par 2 sous forme de 2 colonnes et 10 lignes :

x	2x
0	 0
1	 2
2	 4
3	 6
4	 8
5	 10
6	 12
7	 14
8	 16
9	 18

Décrire d’abord le corps de la page HTML correspondant à la présentation de cette table.

Ecrire ensuite un script Shell envoyant cette page, en utilisant une variable Shell i, qu’on incrémentera avec l’instruction i=$((i+1)), et sur laquelle on bouclera grâce à la structure

while [ $i -lt 10 ]
do
 ...
done
Solution Télécharger mult2.txt 
#!/bin/sh 
echo 'Content-Type: text/html; charset=utf-8'
echo
echo '<html><head>'
echo '<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />'
echo '<title>Tables de multiplication par 2</title>'
echo '</head><body>' 
echo '<table summary="Table de multiplication par 2 sur 2 colonnes">'
echo '<caption>Table de 2</caption>'
echo '<tr><th>x</th><th>2x</th></tr>'
i=0
while [ $i -lt 10 ]
do
echo "<tr><td>" $i "</td><td>" $(($i*2)) "</td></tr>" 
i=$((i+1))
done
echo '</table></body></html>'

6 La stylisation des TD

Pour améliorer la lisibilité d’une table, on peut alterner la couleur de fond de chaque ligne. Ceci s’obtient, entre autres solutions, en plaçant l’attribut style dans les balises tr. La valeur de cet attribut pourrait être par exemple background-color:#777 pour les lignes paires, et background-color:#111 pour les lignes impaires (on étudiera plus tard toutes les possibilités de cet attribut et ses congénères).

Pour savoir si une variable Shell a pour valeur un nombre pair, on peut utiliser l’opérateur % pour calculer son modulo, et utiliser le prédicat -eq pour le comparer à 0. Modifier le script écrit à la fonction précédente pour obtenir la table dans la présentation demandée.

Solution Télécharger mult2_smart.txt 
#!/bin/sh 
echo 'Content-Type: text/html; charset=utf-8'
echo
echo '<html><head>'
echo '<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />'
echo '<title>Tables de multiplication par 2</title>'
echo '</head><body>' 
echo '<table summary="Table de multiplication par 2 sur 2 colonnes">'
echo '<caption>Table de 2</caption>'
echo '<tr><th>x</th><th>2x</th></tr>'
i=0
while [ $i -lt 10 ]
do
if [ $((i % 2)) -eq 0 ]
then
color='#777'
else
color='#111'
fi
echo "<tr style='background-color:$color'><td>$i</td><td>" 
echo $(($i*2))
echo "</td></tr>" 
i=$((i+1))
done
echo '</table></body></html>'

7 La Division des TD

Ecrire un script Shell présentant la même table de multiplication qu’à l’exercice précédent, mais en 10 colonnes et 2 lignes.

Solution Télécharger div2.txt 
#!/bin/sh 
echo 'Content-Type: text/html; charset=utf-8'
echo
echo '<html><head>'
echo '<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />'
echo '<title>Tables de multiplication par 2 sur 10 lignes</title>'
echo '</head><body>' 
echo '<table summary="Table de 2 sur 10 colonnes">'
echo '<caption>Table de 2</caption>'

# Première ligne
echo "<tr><th>x</th>"
i=0
while [ $i -lt 10 ]
do
echo "<td>" $i "</td>"
i=$((i+1))
done
echo "</tr>"

# Deuxième ligne

echo '<tr><th>2x</th>'
i=0
while [ $i -lt 10 ]
do
echo "<td>" $((2*$i)) "</td>"
i=$((i+1))
done
echo "</tr>"
echo '</table></body></html>'
